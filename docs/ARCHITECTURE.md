# Architecture and Data Flow

This document describes the main components of VesselVerse-Framework, how they interact, and where to configure or extend the system.

## Components

- Slicer Extension: `src/slicer_extension/VesselVerse/`

  - `VesselVerse.py`: Main module (UI + logic for loading images/segmentations, editing, saving, and metadata).
  - `loading_dialog.py`, `opacity_slicer.py`: UI helpers.
  - `Resources/Icons/`: Assets.
  - `sql_files/`: SQL scripts used by the extension (populated by scripts when needed).

- Core Library: `src/core/`

  - `dataset.py`: Dataset abstraction; builds paths per model under a base dataset directory.
  - `staple.py`: STAPLE utilities.

- Model/Dataset Registry: `src/model_config/`

  - `model_config.py`: Registers datasets (name, base_path, image_dir, supported models) and models.
  - Where to change dataset roots: edit `base_path` fields here.

- Tracking: `src/tracking/`

  - `track_segmentations.py`: Tracks segmentation versions/paths and relative paths under the dataset root.
  - `verify_tracking.py`: Verifies tracked files and produces reports.

- Orchestration Scripts: `scripts/`

  - `launch_slicer.sh`: Starts Slicer with the VesselVerse module path from `config.sh`.
  - `activate_owner_mode.sh`: Configures DVC remotes (Google Drive) using values from `config.sh`.
  - `install_slicer.sh`, `WSL_*.sh`: Install/launch helpers for macOS/WSL.

- Configuration: `config.sh`

  - `SLICER_PATH`: Path to the Slicer executable (macOS: `/Applications/Slicer.app/Contents/MacOS/Slicer`).
  - `DATASET_NAME`: Dataset key used by DVC helper scripts (e.g., `IXI`, `COW23MR`).
  - `database_ID`, `user_upload_ID`: Google Drive folder IDs.
  - `owner_auth_path`, `user_auth_path`: Service account JSON credentials.

- Example Data Layout: `VESSELVERSE_DATA_IXI/data/`
  - `IXI_TOT/`: Input images (NIfTI).
  - Model outputs: `STAPLE/`, `STAPLE_base/`, `StochasticAL/`, `nnUNet/`, `A2V/`, `Filtering/`.
  - Expert outputs: `ExpertAnnotations/`, `ExpertVAL/`.
  - `metadata/`: JSON metadata generated by tools or Slicer logic.

## Data Flow (Happy Path)

1. Dataset selection

   - UI queries `dataset_registry` from `src/model_config/model_config.py`.
   - Retrieves `base_path`, `image_dir`, and `supported_models` for the selected dataset.

2. Image loading

   - NIfTI is loaded from `${base_path}/${image_dir}/...` (e.g., `IXI_TOT/IXI*-MRA.nii.gz`).
   - Slicer creates/updates scene nodes.

3. Segmentation loading

   - For a chosen model (e.g., `STAPLE`), the module resolves the segmentation path under `${base_path}/STAPLE/...`.

4. Editing and saving

   - Edits occur in Slicerâ€™s Segment Editor.
   - On save, the module exports to a binary labelmap and writes to:
     - `${base_path}/ExpertAnnotations/<imageID>/<imageID>_expert_<expertID>_<timestamp>.nii.gz`
   - A metadata entry is added/updated in:
     - `${base_path}/metadata/<MODEL>_expert_metadata.json`

5. Optional remote sync (DVC)
   - `scripts/activate_owner_mode.sh` configures remotes `storage` and `uploads` using `config.sh` values.
   - Use `dvc push -r uploads` to sync expert outputs to Google Drive.

## Configuration Points

- Slicer binary: `config.sh:SLICER_PATH` (macOS default shown above).
- Dataset roots: `src/model_config/model_config.py` (`base_path` fields).
- DVC remotes: `scripts/activate_owner_mode.sh` (reads IDs/creds from `config.sh`).
- Slicer Additional Module Paths: see `scripts/launch_slicer.sh` or set via Slicer GUI settings.

## Extending the System

- Add a dataset

  - Create/modify a `DatasetConfig` in `model_config.py` and set `base_path`, `image_dir`, `supported_models`.

- Add a model output folder

  - Add the folder name to `supported_models` and ensure the path exists under the dataset root.

- Customize saving or metadata
  - Extend methods in `VesselVerse.py` that export and write metadata.

## Common Pitfalls & Tips

- Slicer not launching module

  - Verify `SLICER_PATH` and that `launch_slicer.sh` passes the correct `--additional-module-paths`.

- Dataset not appearing or wrong folder

  - Confirm `base_path` in `model_config.py` resolves to the correct absolute path.

- `pip` not found on macOS

  - Use `python3 -m pip install ...`.

- DVC not found or remote fails
  - Install DVC with Google Drive support: `python3 -m pip install "dvc[gdrive]"`.
  - Re-run `scripts/activate_owner_mode.sh` and verify with `dvc remote list`.
